# Alert Bot

Телеграм-бот для мониторинга определенного канала и автоматической публикации алертов при обнаружении ключевых фраз.

## Возможности

- Постоянный мониторинг исходного канала на наличие новых постов
- Фильтрация по ключевым фразам (точное совпадение всего текста)
- Игнорирование эмодзи и регистра при сравнении
- Публикация заранее настроенного статичного шаблона при обнаружении совпадения
- Защита от дубликатов (отслеживание обработанных постов)
- Настраиваемый интервал проверки
- Детальное логирование всех операций

## Принцип работы

1. Бот каждые N секунд проверяет новые посты в исходном канале
2. Для каждого нового поста сравнивает его текст с ключевыми фразами
3. При сравнении игнорируются:
   - Эмодзи (🚨, ⚠️, и т.д.)
   - Регистр букв (ТРЕВОГА = тревога)
   - Лишние пробелы
4. Если текст поста **точно** совпадает с одной из ключевых фраз - публикуется алерт
5. Алерт - это **статичный заранее настроенный текст** (не копия исходного поста)
6. Каждый пост обрабатывается только один раз

### Пример работы фильтрации

Ключевая фраза в файле: `Воздушная тревога`

**Будут распознаны:**
- `Воздушная тревога` ✅
- `🚨 Воздушная тревога` ✅
- `ВОЗДУШНАЯ ТРЕВОГА` ✅
- `  воздушная   тревога  ` ✅

**НЕ будут распознаны:**
- `Воздушная тревога в Киеве` ❌
- `Внимание! Воздушная тревога` ❌
- `Объявлена воздушная тревога` ❌

## Установка

### 1. Перейдите в папку проекта

```bash
cd /home/tarrri/projects/bots/alert_bot/
```

### 2. Создайте виртуальное окружение

```bash
python3 -m venv venv
source venv/bin/activate
```

### 3. Установите зависимости

```bash
pip install -r requirements.txt
```

### 4. Настройте конфигурацию

Скопируйте `.env.example` в `.env`:

```bash
cp .env.example .env
```

Отредактируйте `.env` файл и укажите свои данные:

```env
# Telegram API (получить на https://my.telegram.org/apps)
API_ID=12345678
API_HASH=abcdef1234567890abcdef1234567890

# Ваш номер телефона (с кодом страны)
PHONE=+79991234567

# Канал для мониторинга (откуда парсить)
SOURCE_CHANNEL=@air_alert_ua

# Ваш канал (куда публиковать алерты)
TARGET_CHANNEL=@my_alert_channel

# Интервал проверки в секундах
CHECK_INTERVAL=60

# Путь к файлу с ключевыми фразами
KEYWORDS_FILE=data/keywords.txt

# Шаблон алерта (используйте \n для переносов строк)
ALERT_TEMPLATE=🚨 Воздушная тревога!\nСрочно в укрытие!
```

#### Как получить API_ID и API_HASH:

1. Перейдите на https://my.telegram.org/apps
2. Войдите с помощью номера телефона
3. Создайте новое приложение
4. Скопируйте `api_id` и `api_hash`

### 5. Настройте ключевые фразы

Отредактируйте файл `data/keywords.txt` и укажите фразы для отслеживания (по одной на строку):

```text
# Файл с ключевыми фразами
# Строки, начинающиеся с #, игнорируются

Воздушная тревога
Отбой тревоги
Ракетная опасность
Повітряна тривога
Відбій тривоги
```

**Важно:**
- Каждая фраза на отдельной строке
- Пустые строки игнорируются
- Строки с # - комментарии
- При сравнении будут игнорироваться эмодзи и регистр

## Настройка

### Изменение интервала проверки

Отредактируйте `CHECK_INTERVAL` в файле `.env` (значение в секундах):

```env
CHECK_INTERVAL=30  # Проверять каждые 30 секунд
```

### Изменение шаблона алерта

Отредактируйте `ALERT_TEMPLATE` в файле `.env`:

```env
ALERT_TEMPLATE=⚠️ УВАГА! Повітряна тривога!\n\nТермінво в укриття!\n\n#тривога #alert
```

**Советы:**
- Используйте `\n` для переноса строк
- Можно использовать эмодзи
- Можно добавлять хештеги

### Добавление/изменение ключевых фраз

1. Отредактируйте файл `data/keywords.txt`
2. Перезапустите бота

## Часто задаваемые вопросы

### Как работает фильтрация?

Бот сравнивает **весь текст поста целиком** с каждой ключевой фразой из файла. При этом:
- Удаляются все эмодзи
- Текст приводится к нижнему регистру
- Нормализуются пробелы

Если после обработки текст поста **полностью** совпадает с одной из фраз - публикуется алерт.

### Можно ли публиковать оригинальный пост вместо шаблона?

Нет, бот специально разработан для публикации статичного шаблона. Это сделано для:
- Единообразия алертов
- Контроля за форматом сообщений
- Отсутствия лишней информации

### Можно ли мониторить несколько каналов?

В текущей версии - один исходный канал. Для мониторинга нескольких каналов нужно запустить несколько экземпляров бота с разными конфигурациями.

### Что делать, если бот не публикует алерты?

1. Проверьте логи в папке `logs/`
2. Убедитесь, что в `data/keywords.txt` есть ключевые фразы
3. Проверьте, что `ALERT_TEMPLATE` настроен в `.env`
4. Убедитесь, что бот имеет права администратора в целевом канале
5. Проверьте точность совпадения фразы (без лишних слов)

### Как протестировать бота?

1. Создайте тестовый канал
2. Укажите его в `TARGET_CHANNEL`
3. Добавьте тестовую фразу в `data/keywords.txt`
4. Опубликуйте пост с этой фразой в исходном канале
5. Проверьте, что алерт появился в целевом канале

- Никогда не коммитьте `.env` файл в git
- Храните `API_HASH` в секрете
- Session файлы содержат данные авторизации - не передавайте их третьим лицам
- Не публикуйте токены и ключи доступа
